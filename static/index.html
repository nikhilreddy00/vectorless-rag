How Vectorless RAG Works

1

Tree Structure

SEC filings are indexed into a hierarchical JSON tree — like a smart table of contents with summaries. No vector embeddings needed.

2

LLM Reasoning

When you ask a question, Claude reads the tree structure and reasons about which sections contain the answer — just like a human would.

3

Targeted Retrieval

Only the relevant sections are fetched and sent to the LLM — no similarity search, no noisy top-K results.

4

Grounded Answer

The final answer is generated solely from the retrieved sections, with precise citations to specific items and line numbers.
